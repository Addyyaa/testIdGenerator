# 故障排除指南

## 常见警告和错误

### 1. punycode 弃用警告

**现象：**
```
(node:xxxx) [DEP0040] DeprecationWarning: The `punycode` module is deprecated.
```

**原因：**
这个警告来自 Node.js 的依赖项（通常是 VS Code 扩展主机进程或其他依赖项），不是本扩展代码的问题。这是 Node.js 的一个已知弃用警告，不影响扩展功能。

**解决方案：**
- 这个警告是**无害的**，可以安全忽略
- 扩展代码已经尝试抑制这个警告，但如果警告在扩展加载之前触发，可能仍然会显示
- 这是 VS Code/Cursor 内部的问题，等待 VS Code 更新即可解决

### 2. NoWorkspaceUriError 错误

**现象：**
```
repoResult.error NoWorkspaceUriError
```

**原因：**
当文件不在工作区中打开时，VS Code 可能会触发这个错误。这是 VS Code 内部的错误，不是本扩展的问题。

**解决方案：**
- 扩展代码已经添加了完善的错误处理，会自动处理这种情况
- 这个错误**不影响扩展功能**，扩展可以在没有工作区的情况下正常工作
- 如果需要在工作区中使用扩展，请先打开一个文件夹作为工作区

## 扩展功能说明

本扩展设计为在以下情况下都能正常工作：

1. ✅ 有工作区文件夹时
2. ✅ 没有工作区文件夹时（使用全局配置）
3. ✅ 文件不在工作区中时
4. ✅ 单个文件打开时

所有错误情况都已经被妥善处理，扩展会自动降级到默认配置继续工作。

## 如果问题仍然存在

如果遇到其他问题：

1. 检查 VS Code/Cursor 版本是否是最新的
2. 重新加载扩展窗口（`Cmd+Shift+P` -> `Developer: Reload Window`）
3. 检查扩展是否正确激活（查看输出面板中的扩展日志）
4. 如果问题持续，请提交 Issue 并附上完整的错误日志
